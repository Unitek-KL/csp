FLEXCOMx_USART_TransmitterIsReady Function

This function returns the hardware status associated with the Transmit hardware FIFO of the given FLEXCOM USART peripheral instance. 

When Transmitter is ready, user can submit data to transmit. 

This function is available only in non-interrupt mode of operation. And can be used to achieve non-blocking behavior of write request. User has to check the Transmit ready status by calling this function and can submit write request for the buffer size of 1.

C
bool FLEXCOMx_USART_TransmitterIsReady();
Preconditions

FLEXCOMx_USART_Initialize must have been called for the associated FLEXCOM USART instance.

Returns

true - Transmit hardware FIFO has empty space to accept the data. false - Transmit hardware FIFO is full.

Remarks

If user submits write request for more than 1 byte after checking the ready status, then write API will block until the submitted length of data is processed. This is the native behavior of the write API in the non-interrupt or blocking mode of the library.

Example
if(true == FLEXCOMx_USART_TransmitterIsReady())
{
    // Empty space is available in Transmitter FIFO, hence can write a byte
    FLEXCOMx_USART_Write(&txData, 1);
}
else
{
    //Transmitter is busy
}
