adc_dma_sequencing

This example shows how to use DMA sequencing feature available in ADC peripheral library. 

This feature enables user to reconfigure the ADC using DMA. The example application uses the DMA sequencing feature to write to MUXPOS bit field in the INPUTCTRL register to select input pin before an ADC conversion. Two DMA channels are used in this application. One channel to copy the results from ADC result register to an array in SRAM, another DMA channel to copy new MUXPOS value to the INPUTCTRL register. The ADC DMA sequencing trigger is used as trigger source for the second DMA channel which updates INPUTCTRL. The RTC peripheral acts as a trigger to start ADC conversion. ADC data is printed on the USART console via EDBG Virtual COM Port in hexadecimal format.
