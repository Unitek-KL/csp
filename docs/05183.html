SERCOMx_USART_ReceiverIsReady Function

This function returns the hardware status associated with the Receive hardware FIFO of the given USART peripheral instance. When Receiver is ready, user can read the available data. 

This function is available only in non-interrupt mode of operation. It can be used to achieve non-blocking behavior of SERCOMx_USART_Read() function even if the library is configured for blocking behavior. If this function returns true, calling the SERCOMx_USART_Read() function with a byte count of 1 will result in non-blocking behavior.

C
bool SERCOMx_USART_ReceiverIsReady();
Preconditions

SERCOMx_USART_Initialize must have been called for the associated USART instance.

Returns
  • true - if receive hardware FIFO has at least one data available to read.
  • false - if receive hardware FIFO is empty.
Remarks

If user submits read request for more than 1 byte after checking the ready status, then read API will block until the submitted length of data is processed. This is the native behavior of the read API in the non-interrupt or blocking mode of the library.

Example
char rxData;
if(true == SERCOM0_USART_ReceiverIsReady())
{
    // At least one data is available in Receive FIFO, hence can read a byte
    SERCOM0_USART_Read(&rxData, 1);
}
else
{
    //Receiver is empty
}
