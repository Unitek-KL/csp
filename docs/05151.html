SERCOMx_SPI_IsBusy Function

This function returns true if the SERCOM x SPI module is busy with a transfer. The application can use the function to check if SERCOM x SPI module is busy before calling any of the data transfer functions. The library does not allow a data transfer operation if another transfer operation is already in progress. 

This function can be used as an alternative to the callback function when the library is operating interrupt mode. The allow the application to implement a synchronous interface to the library.

C
bool SERCOMx_SPI_IsBusy();
Preconditions

The SERCOMx_SPI_Initialize() should have been called once. The module should have been configured for interrupt mode operation in MHC.

Returns
  • true - Transfer is still in progress
  • false - Transfer is completed or no transfer is currently in progress.
Remarks

None.

Example
    // The following code example demonstrates the use of the
    // SERCOMx_SPI_IsBusy() function. This example shows a blocking while
    // loop. The function can also be called periodically.

    uint8_t dataBuffer[20];

    SERCOM1_SPI_Initialize();
    SERCOM1_SPI_Write(dataBuffer, 20);

    while (SERCOM1_SPI_IsBusy() == true)
    {
        // Wait here till the transfer is done.
    }
